## What This Repo Is

Documentation and KiCad schematic files for a custom LED lighting system in a Warhammer 40K Reaver Titan — Emperor's Children chaos build. All LEDs are green. The model stands on a destroyed tank.

## Tooling Commands

```bash
# Generate/update CHANGELOG.md from conventional commit history
npm run changelog
```

Commits must follow [Conventional Commits](https://www.conventionalcommits.org/) — enforced by commitlint + husky on pre-commit. There are no build steps, linting commands, or tests beyond this.

## GitHub Workflow

When creating GitHub issues, always add them to the **Reaver Titan Build** project (project number 2, owner `learnsometing`):

```bash
# Create issue
gh issue create --title "..." --body "..."

# Add to project
gh project item-add 2 --owner learnsometing --url <issue-url>
```

After making changes, commit and push, then open a PR against `main` if one isn't already open:

```bash
git add <files>
git commit -m "type: description"
git push -u origin <branch>

# Open PR if none exists for this branch
gh pr create --title "..." --body "..." --base main
```

## Repository Structure

- `docs/` — One markdown file per lighting zone plus a schematic tracker. These are the authoritative design references for electrical decisions, component specs, and animation behavior.
- `kicad/` — KiCad 10 project (`reaver-titan-lighting.*`). Single schematic file with multiple hierarchical sheets; ERC reports are stored here.
- `CHANGELOG.md` — Generated by changelogen; do not edit manually.

## Key Electrical Decisions

- **MOSFET:** IRLML6344 (SOT-23) for all high-current LED rope channels — fully switches at 3.3V gate
- **Drive circuit:** 10kΩ gate pull-down to GND; series resistor (6.8Ω 1W) between +5V and LED rope anode
- **PCA9685 limit:** 25mA per channel — any load above that requires MOSFET switching
- **Wire gauge:** 14 AWG on 7.4V side, 22 AWG on 5V side
- **Buck converter:** Set to 5V before connecting anything — adjustable via potentiometer
- `docs/core-infrastructure.md` and `docs/blaster.md` are the primary references for component specs and circuit patterns
