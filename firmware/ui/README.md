# Reaver Titan Web UI

Control interface served directly from the ESP32-S3 over WiFi. No router required — the ESP32 hosts its own access point and serves this UI as a single HTML file from flash memory.

## Aesthetic

Dark Mechanicus terminal. Green phosphor on black, scanline overlays, glitch effects on startup, gothic tech typography. The UI should feel like interfacing with the machine spirit of the titan itself.

## Constraints

- **Must bundle to a single HTML file.** The ESP32 WebServer serves one file from a PROGMEM string — no filesystem, no asset loading. All JS, CSS, and fonts must be inlined at build time.
- **Bundle size matters.** Smaller is better. Target under 100KB uncompressed.
- **No internet connection.** The device has no internet access, so no CDN imports or external font URLs.

## Stack

- **Svelte + Vite** — Svelte compiles away at build time (no runtime shipped), producing clean flat JS. Vite handles bundling, inlining, and hot reload during development.

## Zone Map

| Zone | Type | UI Control |
|------|------|------------|
| Head | Static | On/off toggle |
| Torso | Static | On/off toggle |
| Blaster | Animated, multi-state | State controller (see below) |

### Blaster States

The blaster boots into IDLE and transitions on demand via the UI.

| State | Description |
|-------|-------------|
| IDLE | Faint glow in rear heatsink only. Default on power-up. |
| PRIME | Energy propagates forward through heatsink zones sequentially. |
| FULL CHARGE | All heatsink zones at full brightness, barrels ramping up, subtle oscillation. |
| DISCHARGE | Barrels flash, fast rear-to-front pulse, fades back to IDLE. |

## UI Structure

Single page app with client-side view switching (show/hide — no router, no additional pages served from the ESP32).

- **Overview** — zone status at a glance, head/torso toggles, weapon cards
- **Blaster view** — state controller, fire trigger button; accessed by tapping the blaster weapon card
- Back button returns to overview

Additional weapon views follow the same pattern as the blaster when added.

## Dev Workflow

```bash
cd firmware/ui
npm install
npm run dev       # local dev server with hot reload
```

A mock server mimics the ESP32's HTTP endpoints so the UI can be developed fully in the browser without hardware. See endpoint list below.

```bash
npm run build     # bundles and generates ui.h
```

`ui.h` defines `INDEX_HTML` as a PROGMEM string and is `#include`d by the Arduino sketch. Do not edit `ui.h` manually — it is generated by the build.

## HTTP Endpoints

| Method | Path | Params | Description |
|--------|------|--------|-------------|
| GET | `/` | — | Serves the UI |
| POST | `/zone/head` | `state=on\|off` | Toggle head zone |
| POST | `/zone/torso` | `state=on\|off` | Toggle torso zone |
| POST | `/blaster/state` | `state=idle\|prime\|charge\|discharge` | Set blaster state |
